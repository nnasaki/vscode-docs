# November 2025 Insiders (version 1.107)

![VS Code Insiders banner](images/1_107/vscode-insiders-banner-medium.png)

_最終更新日: 2025年12月3日_

これらのリリースノートは VS Code の Insiders ビルドをカバーしており、新機能が追加されるにつれて進化し続けます。最新のアップデートを試すには、[Insiders をダウンロード](https://code.visualstudio.com/insiders)してください。
これらのリリースノートをオンラインで読むには、[code.visualstudio.com/updates](https://code.visualstudio.com/updates) にアクセスしてください。

[コミットログ](https://github.com/Microsoft/vscode/commits/main)と[クローズされた課題](https://github.com/Microsoft/vscode/issues?q=is%3Aissue+is%3Aclosed+milestone%3A%22November+2025%22)のリストで進捗を追跡できます。

_これらのリリースノートは GitHub Copilot を使用して生成されており、不正確な情報が含まれている可能性があります。_

Happy Coding!

## December 3, 2025

* <span class="tag tag-git">git</span> スタッシュのタイムスタンプが、ソース管理リポジトリビューのスタッシュ項目の右側に表示されるようになりました。これにより、ブランチやタグのタイムスタンプ表示と一貫した配置と読みやすさが向上しました。同一の相対タイムスタンプを持つ連続したスタッシュは、重複テキストの代わりに控えめな縦線インジケーターを表示し、ホバーすると完全なタイムスタンプが表示されます。_[#280608](https://github.com/microsoft/vscode/issues/280608)_

## December 2, 2025

* <span class="tag tag-chat">chat</span> 新しい `chat.restoreLastPanelSession` 設定を使用して、VS Code 再起動時のアクティブなチャットセッションの復元を無効にできるようになりました。この設定により、VS Code を開いたときにクリーンなチャット状態で開始でき、以前のチャット会話の自動復元を防ぐことができます。_[#280402](https://github.com/microsoft/vscode/issues/280402)_

* <span class="tag tag-prompts">prompts</span> <span class="tag tag-agents">agents</span> プロンプト、エージェント、および指示ファイルが、手動でアクティベーション（Ctrl+Space）を行わなくても、フロントマターのトップレベルエントリに対して IntelliSense の提案を自動的にトリガーするようになりました。これにより、これらのファイルを作成する際に利用可能なオプションを発見して設定することが容易になります。_[#279466](https://github.com/microsoft/vscode/issues/279466)_

* <span class="tag tag-chat">chat</span> セッション処理機能が強化され、チャットモデルのライフサイクル管理が改善されました。チャットモデルがチャットウィジェットとは独立して存在できるようになり、バックグラウンドセッションの作成をサポートし、応答の進行状況と確認状態を監視するためのより明確な observable 状態を提供することで、全体的なチャットセッションの信頼性とパフォーマンスが向上しました。_[#277318](https://github.com/microsoft/vscode/issues/277318)_

* <span class="tag tag-nes">NES</span> <span class="tag tag-copilot">copilot</span> Next Edit Suggestions が、新しい「ジャンプ先」コントロールを使用して、同じファイル内で現在のカーソル位置からより離れた場所にある予測を表示するようになりました。このコントロールは次の予測編集の場所を示し、その位置にすばやく移動して提案を受け入れることができます。_[#277816](https://github.com/microsoft/vscode/issues/277816)_

## December 1, 2025

* <span class="tag tag-agents">agents</span> `.github-private` リポジトリを通じて、Enterprise/Organization レベルで共有されているカスタムエージェントにアクセスできるようになりました。組織の `.github-private/agents` フォルダーに設定されたカスタムエージェントが VS Code に表示されるようになり、Web 版の機能と一致します。_[#274341](https://github.com/microsoft/vscode/issues/274341)_

* <span class="tag tag-git">git</span> ソース管理ビューのタグメニューにある新しい **Push Tags** オプションを通じて、Git タグを直接プッシュできるようになりました。コマンドパレットを使用せずに、タグ管理がより簡単にアクセスできるようになりました。_[#279793](https://github.com/microsoft/vscode/issues/279793)_

* <span class="tag tag-chat">chat</span> 新しい `chat.welcome.enabled` 設定を使用して、空のチャットウィンドウのウェルカムバナーとアイコンを非表示にできるようになりました。空のチャットで右クリックすると、コンテキストメニューからこの設定を切り替えることができます。_[#280287](https://github.com/microsoft/vscode/issues/280287)_

* <span class="tag tag-typescript">TypeScript</span> バージョンピッカーから実験的な TypeScript ネイティブバージョンを選択できるようになりました。**Select TypeScript Version** コマンドには、利用可能な場合に実験的な TypeScript ネイティブのオプションが含まれ、実験的な TypeScript 機能を発見して使用することが容易になりました。_[#280206](https://github.com/microsoft/vscode/issues/280206)_

* <span class="tag tag-agents">agents</span> エージェントモードが、組み込みツールが無効になっている場合に `listDirectory` ツールを適切に尊重するようになりました。以前は、検索ツールと `listDirectory` ツールがオフになっていても、ディレクトリ構造とファイル名が LLM に渡されていました。_[#280183](https://github.com/microsoft/vscode/issues/280183)_

* <span class="tag tag-git">git</span> <span class="tag tag-copilot">copilot</span> Git 拡張機能が、Git ワークツリー用の Copilot CLI シナリオをサポートする API を提供するようになりました。これにより、ワークツリーの作成、リポジトリとワークツリー間の変更の移行、およびワークツリーの削除が可能になります。_[#277163](https://github.com/microsoft/vscode/issues/277163)_

* <span class="tag tag-copilot">copilot</span> VS Code が Claude Skills ファイル（`.claude/skills/<skill-name>/SKILL.md`）を読み取れるようになりました。これにより、Claude Code と GitHub Copilot の両方を使用する組織が、VS Code で GitHub Copilot と既存の Claude Skills を再利用できるようになります。_[#275621](https://github.com/microsoft/vscode/issues/275621)_

* <span class="tag tag-git">git</span> ソース管理同期インジケーターがソース管理ビューで常に表示されるようになりました。アイコンにホバーすると、ツールチップに同期ステータス（受信/送信の変更）が表示されます。_[#275303](https://github.com/microsoft/vscode/issues/275303)_

## November 27, 2025

* <span class="tag tag-chat">chat</span> URL ハンドラーを使用してチャットプロンプトテキストを設定できるようになりました。チャット URL ハンドラーが新しい `prompt` クエリパラメーターをサポートし、VS Code を開く際に外部アプリケーションがチャット入力フィールドを事前入力できるようになりました。_[#279701](https://github.com/microsoft/vscode/issues/279701)_

## November 26, 2025

* <span class="tag tag-git">git</span> Git リポジトリエクスプローラービューに **Stashes** ノード（`scm.repositories.explorer` 設定が必要）が含まれるようになりました。相対タイムスタンプとブランチ名を含むすべてのスタッシュが表示されます。コンテキストメニューから直接スタッシュの作成、表示、適用、ポップ、ドロップができます。_[#279605](https://github.com/microsoft/vscode/issues/279605)_

* <span class="tag tag-mcp">MCP</span> fetch ツールが、クライアントサイドでレンダリングされるページ（React アプリなど）がコンテンツを抽出する前に完全に読み込まれるまで待機するために、ネットワークトラフィック追跡ヒューリスティックスを使用するようになりました。これにより、非同期でデータをフェッチする動的なウェブサイトの信頼性が向上します。_[#248453](https://github.com/microsoft/vscode/issues/248453)_

* <span class="tag tag-mcp">MCP</span> fetch ツールが、エラーレスポンス（404 など）を 24 時間ではなく 5 分間のみキャッシュするようになりました。これにより、以前利用できなかったページが利用可能になったことを、完全なキャッシュ有効期限を待たずに検出できるようになりました。_[#268104](https://github.com/microsoft/vscode/issues/268104)_

* <span class="tag tag-chat">chat</span> 新しいインラインチャット UI がすべてのユーザーにデフォルトで有効になりました。`inlineChat.enableV2` 設定はバックエンドの動作のみを制御し、現在のエディター外で変更されたファイルはサイドエディターグループで自動的に開かれます。_[#278054](https://github.com/microsoft/vscode/issues/278054)_

## November 25, 2025

* <span class="tag tag-agents">agents</span> エージェントビューが、実行中のエージェントセッションの詳細な進行状況情報を表示するようになりました。一般的な「working...」メッセージの代わりに、最後のツール呼び出しの進行状況を表示し、特にセッション実行中にエディターが閉じているときに、エージェントが現在何をしているかの可視性が向上しました。_[#278469](https://github.com/microsoft/vscode/issues/278469)_

## November 24, 2025

* <span class="tag tag-ui">UI</span> Windows 上の VS Code が、Chromium のアップデーターに触発されたバージョン管理されたリソース更新メカニズムを使用するようになりました。これにより、アトミックなインプレース置換の代わりにバージョン管理されたパッケージパスを使用することで、更新の信頼性が大幅に向上します。これは、以前のアップデーターが処理できなかった OS の強制シャットダウンなどのエッジケースでの壊れたインストールを防ぐのに役立ちます。_[#249239](https://github.com/microsoft/vscode/issues/249239)_

## November 23, 2025

* <span class="tag tag-agents">agents</span> <span class="tag tag-copilot">copilot</span> 新しい Copilot CLI エージェントセッションを作成するためのコマンドパレットエントリとエディターツールバーボタンが追加されました。ローカルチャットエージェントのワークフローと一致し、エージェントセッションビューをナビゲートする必要がなくなりました。_[#273441](https://github.com/microsoft/vscode/issues/273441)_

## November 22, 2025

* <span class="tag tag-chat">chat</span> <span class="tag tag-terminal">terminal</span> チャットターミナル出力が、プレーンな HTML テキストの代わりに埋め込み xterm.js ターミナルインスタンスを使用してレンダリングされるようになりました。適切なターミナルフォーマット、ANSI カラーサポート、およびコマンド出力の正確なテキストレンダリングを提供します。_[#276946](https://github.com/microsoft/vscode/issues/276946)_

## November 21, 2025

* <span class="tag tag-authentication">authentication</span> Microsoft 認証が、オプトインした登録済みマシンで Linux x64 上の MSAL 認証ブローカーをサポートするようになりました。Linux プラットフォームで強化された認証セキュリティとコンプライアンス機能が有効になります。_[#278688](https://github.com/microsoft/vscode/issues/278688)_

* <span class="tag tag-authentication">authentication</span> Microsoft 認証拡張機能が、Intel ベースの Mac で MSAL ブローカー認証をサポートするようになりました。ブローカーサポートにオプトインした登録済みデバイスのシステムレベルの資格情報管理を通じて、セキュリティが強化されます。_[#278687](https://github.com/microsoft/vscode/issues/278687)_

* <span class="tag tag-ui">UI</span> コマンドパレットの最近使用したコマンドを、各最近使用したコマンドの横に表示される「X」ボタンを使用して個別に削除できるようになりました。他のコマンドセクションに影響を与えることなく、コマンド履歴をより良く整理できます。_[#277311](https://github.com/microsoft/vscode/issues/277311)_

* <span class="tag tag-chat">chat</span> カスタムチャットモードがキーボードショートカットをサポートするようになりました。キーボードショートカットエディターを通じて、拡張機能によって提供されるチャットモードのキーバインディングを設定でき、以前は組み込みモードでのみ利用可能だった機能と一致します。_[#270662](https://github.com/microsoft/vscode/issues/270662)_

* <span class="tag tag-chat">chat</span> <span class="tag tag-agents">agents</span> チャットセッションビューが、異なるプロバイダーからのすべてのチャットおよびエージェントセッションを 1 か所に表示する統合シングルビューモードをサポートするようになりました。フィルタリング、検索、およびセッション状態を一目で管理するための強化された機能を備えています。_[#270945](https://github.com/microsoft/vscode/issues/270945)_

* <span class="tag tag-ui">UI</span> 「最近開いた」ウィンドウピッカーが、他のウィンドウで既に開いているワークスペースやフォルダーを視覚的にハイライト表示するようになりました。複数のプロジェクトを同時に作業する際に、既存のインスタンスを特定して切り替えることが容易になりました。_[#268277](https://github.com/microsoft/vscode/issues/268277)_

* <span class="tag tag-ui">UI</span> 現在アクティブなウィンドウ以外のすべての VS Code ウィンドウを閉じることができる新しい「他のウィンドウを閉じる」コマンドが追加されました。複数のインスタンスを開いて作業する際のウィンドウ管理が改善されました。_[#233635](https://github.com/microsoft/vscode/issues/233635)_

* <span class="tag tag-mcp">MCP</span> fetch ツールが、きめ細かい URL 承認オプションをサポートするようになりました。特定の URL またはドメインに基づいてフェッチを自動的に承認できるようになりました（例：「`bing.com` からの URL を常に承認する」）。以前のセッション/ワークスペースレベルの権限の代わりに、ターミナルコマンドの自動承認と同様のより細かい制御を提供します。_[#265850](https://github.com/microsoft/vscode/issues/265850)_

## November 20, 2025

* <span class="tag tag-mcp">MCP</span> <span class="tag tag-copilot">copilot</span> 組み込みの GitHub 拡張機能に GitHub MCP サーバーが含まれるようになりました。Copilot がチャットを通じて GitHub リポジトリ、課題、プルリクエスト、および組織と直接やり取りするためのツールを提供します。_[#254836](https://github.com/microsoft/vscode/issues/254836)_

* <span class="tag tag-ui">UI</span> Tree-sitter WASM ライブラリが最新バージョンに更新されました。VS Code で使用される言語文法の構文解析パフォーマンスと機能が向上しました。_[#274547](https://github.com/microsoft/vscode/issues/274547)_

* <span class="tag tag-terminal">terminal</span> PowerShell tree-sitter 文法がアップグレードされました。解析の信頼性が向上し、PowerShell 7 の `&&` および `||` 演算子のサポートが追加され、PowerShell 5 と 7 の両方で強化されたコマンド書き換え機能が提供されます。_[#274548](https://github.com/microsoft/vscode/issues/274548)_

## November 19, 2025

* <span class="tag tag-ui">UI</span> VS Code がプライベート拡張機能マーケットプレイス/ギャラリーをサポートするようになりました。企業が内部拡張機能をホストして配布できるようになりました。_[#21839](https://github.com/microsoft/vscode/issues/21839)_

## November 18, 2025

* <span class="tag tag-copilot">copilot</span> Codespaces で VS Code を使用する際に GitHub Enterprise ポリシーが適用されるようになりました。リモート開発環境で組織レベルのセキュリティとコンプライアンス設定が適切に適用されることが保証されます。_[#278127](https://github.com/microsoft/vscode/issues/278127)_

* <span class="tag tag-chat">chat</span> <span class="tag tag-terminal">terminal</span> チャットターミナルツールが、ターミナルコマンドの終了コード、開始時刻、および実行時間を表示するようになりました。エージェントセッション中にスタックしたコマンドや長時間実行されているコマンドを特定しやすくなりました。_[#272575](https://github.com/microsoft/vscode/issues/272575)_

* <span class="tag tag-chat">chat</span> <span class="tag tag-search">search</span> チャットエージェントの `textSearch` ツールが、コードを検索する際に `search.useIgnoreFiles` 設定を尊重するようになりました。エージェントによって明示的にクエリされた場合、.gitignore やその他の除外されたディレクトリ内のファイルを検索できるようになりました。_[#270857](https://github.com/microsoft/vscode/issues/270857)_

## November 17, 2025

* <span class="tag tag-settings">settings</span> 設定のデフォルトフォーマッタードロップダウンが、インストールされているすべての拡張機能を無差別にリストするのではなく、実際にフォーマット機能を提供する拡張機能をフィルタリングして優先表示するようになりました。_[#277783](https://github.com/microsoft/vscode/issues/277783)_

## November 15, 2025

* <span class="tag tag-ui">UI</span> フォーカスされていないエディターグループで現在の行の背景色を独立して制御するための `editor.inactiveLineHighlightBackground` テーマカラーが追加されました。マルチエディターワークフローでアクティブなエディターと非アクティブなエディターの視覚的な区別が改善されました。_[#277272](https://github.com/microsoft/vscode/issues/277272)_

* <span class="tag tag-terminal">terminal</span> <span class="tag tag-copilot">copilot</span> Copilot によって提案されたターミナルコマンドを、現在のチャットセッションのみで許可できるようになりました。設定を永続的に変更することなく、特定のタスク中にリスクのあるコマンドを実行するためのワークスペースレベルの設定と並んで、一時的な許可オプションを提供します。_[#260819](https://github.com/microsoft/vscode/issues/260819)_

## November 13, 2025

* <span class="tag tag-mcp">MCP</span> MCP サーバーリクエストが、単なる `node` ではなく、適切な User-Agent ヘッダー（例：`VSCode/x.x.x (darwin arm64)`）を含むようになりました。MCP サーバー開発者が、分析、モニタリング、およびデバッグの目的で、VS Code トラフィックを他の Node.js アプリケーションと区別できるようになりました。_[#264502](https://github.com/microsoft/vscode/issues/264502)_

* <span class="tag tag-chat">chat</span> <span class="tag tag-terminal">terminal</span> チャットターミナルセッションにフォーカスして出力を展開するためのキーバインディングを設定できるようになりました。Copilot Chat によって実行されるターミナルコマンドを操作する際のキーボードナビゲーションのアクセシビリティが向上しました。_[#274411](https://github.com/microsoft/vscode/issues/274411)_

## November 12, 2025

* <span class="tag tag-chat">chat</span> チャットメッセージで明示的に言及された URL が自動的に承認されるようになりました。ユーザーが URL をチャットインターフェースに直接貼り付ける際に、手動での URL 承認が不要になりました。_[#276955](https://github.com/microsoft/vscode/issues/276955)_

* <span class="tag tag-terminal">terminal</span> ターミナル出力モニターの「タスク待機」確認ダイアログにキーボードショートカットサポート（Cmd/Ctrl+Enter）が追加されました。他の確認ダイアログとの一貫性が向上し、Copilot がタスクにユーザー入力が必要であることを検出した際のワークフローが高速化されました。_[#276833](https://github.com/microsoft/vscode/issues/276833)_

* <span class="tag tag-terminal">terminal</span> チャットターミナルツールを使用する際にターミナルコマンド出力がシリアライズされて永続化されるようになりました。ターミナルインスタンスが閉じられたり終了されたりした後でも、コマンド結果を表示できるようになりました。_[#274870](https://github.com/microsoft/vscode/issues/274870)_

* <span class="tag tag-copilot">copilot</span> AI 関連のコードアクション（「ドキュメント生成」など）が、クイック修正メニュー（Ctrl/Cmd + .）とライトバルブホバーにデフォルトで直接表示されるようになりました。最初に Copilot 拡張機能をインストールする必要なく、AI 支援機能に即座にアクセスできます。_[#260136](https://github.com/microsoft/vscode/issues/260136)_

* <span class="tag tag-ui">UI</span> コメントスレッドのガターアイコンが、スレッドにドラフトコメントが含まれている場合に個別のドラフトアイコンを表示するようになりました。既存の解決済みおよび未解決のコメントインジケーターと並んで、未公開のフィードバックの可視性が向上しました。_[#179162](https://github.com/microsoft/vscode/issues/179162)_

* <span class="tag tag-mcp">MCP</span> MCP サーバーが、コンテンツを base64 ブロブとして埋め込む代わりに、`https://` URI を使用してリソースを返すことができるようになりました。画像やその他の Web アクセス可能なリソースを提供する際のコンテキストサイズが削減されます。_[#259253](https://github.com/microsoft/vscode/issues/259253)_

## November 11, 2025

* <span class="tag tag-ui">UI</span> クイックオープンピッカーでオフセットモードの発見しやすさを向上させるための新しい **Go To Offset** コマンドが追加されました。これは入力を `::` で事前入力し（Go To Line/Column が `:` で事前入力するのと同様）、ファイル内の特定のバイトオフセットにナビゲートするのに役立ちます。_[#275317](https://github.com/microsoft/vscode/issues/275317)_

* <span class="tag tag-accessibility">accessibility</span> <span class="tag tag-terminal">terminal</span> ターミナルインライン出力がスクリーンリーダーにアクセス可能になりました。スクリーンリーダーモードでは、コンテンツが HTML ではなくテキストとしてレンダリングされ、支援技術に依存するユーザーのナビゲーションと読みやすさが向上しました。_[#274220](https://github.com/microsoft/vscode/issues/274220)_

* <span class="tag tag-copilot">copilot</span> Copilot が機密ファイル（settings.json など）の編集許可を要求する際に、生の JSON ツール呼び出しデータの代わりにインライン差分プレビューが表示されるようになりました。承認前に提案された変更を確認しやすくなりました。_[#266574](https://github.com/microsoft/vscode/issues/266574)_

* <span class="tag tag-ui">UI</span> 「エディターを前のグループに移動」コマンドが、最も左のグループから呼び出された場合（複数のエディターが含まれている場合）に、左側に新しいエディターグループを自動的に作成するようになりました。「エディターを次のグループに移動」の動作を反映し、エディターを配置するためのキーボード駆動のワークフローが改善されました。_[#262817](https://github.com/microsoft/vscode/issues/262817)_

* <span class="tag tag-ui">UI</span> 最近開いたピッカーが、現在他のウィンドウで開いているフォルダーとワークスペースを視覚的に示すようになりました。複数のワークスペースインスタンスを特定して管理しやすくなりました。_[#276460](https://github.com/microsoft/vscode/issues/276460)_

* <span class="tag tag-terminal">terminal</span> ターミナル提案ウィジェットが、引数の提案をオプションやフラグとは別にグループ化するようになりました。提案リストでコマンド引数とフラグを区別しやすくなりました。_[#272352](https://github.com/microsoft/vscode/issues/272352)_
